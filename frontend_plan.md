### F0：工程基座与前后端分离基线

**目标**：Vue 工程跑通；路由骨架、Pinia、Axios、环境变量、目录结构固定。
 **产物**：`frontend-vue/` 工程；`apiClient`；基础 Layout 占位；占位页可访问。
 **验收**：能启动、能路由跳转、axios 能带 token（即使登录接口暂未接也不报错）。

**风格窗口 1（只允许在 F0 末尾做）**：

- **允许做**：建立 `tokens.css`（仅变量层），把 Element Plus 主题变量映射到 tokens；全局基础样式（背景、字体、圆角、阴影等级、间距尺度）。
-  **禁止做**：任何具体页面的“好看优化”、Notion 风细节、逐页改布局。

### F1：登录/鉴权/会话与角色入口

**目标**：登录/退出、token 持久化、路由守卫、按角色的菜单与可访问路由。
 **产物**：`/login`；`/`（layout）；角色 Dashboard 占位。
 **验收**：登录成功进入系统；刷新会话保持；401 自动回登录。

> 风格规则：**禁止做页面美化**，只允许必要的布局修正（如间距）。

### F2：个人中心与技能画像（对齐后端 M1）

**目标**：个人信息展示/更新；技能绑定/解绑闭环。
 **产物**：Profile 页、Skills 页。
 **验收**：list→bind→delete→list 成功闭环。

> 风格规则：禁止逐页美化。

### F3：竞赛列表/检索/详情（对齐后端 M2）

**目标**：确立全站“页面范式”（列表/详情的结构与交互）。
 **产物**：Competitions List + Search/Filter + Detail。
 **验收**：关键词/状态筛选生效；详情字段完整；错误提示统一。

> 风格规则：禁止逐页美化，但允许**统一页面范式的结构定稿**（例如列表用表格/卡片二选一），这是“架构决定”，不是美化。

### F4：教师申请与管理员审核（对齐后端 M3）

**目标**：教师申请→管理员审核通过/拒绝→状态可追踪。
 **产物**：教师申请入口、管理员审核列表。
 **验收**：PENDING→APPROVED/REJECTED 全流程可演示。

> 风格规则：禁止逐页美化。

### F5：学生报名申请与教师审核入队（对齐后端 M4）

**目标**：学生申请加入教师组→教师审批→学生看到入队结果。
 **产物**：申请入口、教师审批页、学生“我的申请/我的组”入口。
 **验收**：申请→审批→成员状态变化在页面体现。

> 风格规则：禁止逐页美化。

### F6：组队控制与成员可见性（对齐后端 M5）

**目标**：成员列表权限、关闭组队、移除成员等控制入口与提示。
 **产物**：Team 页、Members 页、管理操作按钮（按权限显示/隐藏）。
 **验收**：

- 不同角色看到不同能力；403/409 有明确业务提示。
- **补充验收（终止态 DISBANDED）**：
  - 若 team.status == DISBANDED：前端隐藏/禁用所有写操作入口（申请/审核/移除/发帖/提交等），并展示统一提示“队伍已解散，操作已禁止”；
  - 若用户仍通过旧页面/缓存入口触发写操作且后端返回 409（team is disbanded），前端需要将其转为可读提示并引导返回队伍列表。

**风格窗口 2（真正的“统一换皮”）**

- **时间点**：**F6 完成后、进入 F7 之前**

-  **允许做（一次性完成）**：

  - 抽稳定的 Layout（左侧菜单/顶部栏/内容区规范）

  - 抽通用展示组件（SectionCard、StatCard、EmptyState、PageHeader）

  - 基于 `tokens.css` 做全站 Notion 风统一（留白、层级、弱边框、卡片系统）

  - 统一标题体系与间距系统

- **禁止做**：逐页面“手工精修”、大规模自定义组件库、引入第二套 UI 框架。

### F10：奖项发布与个人荣誉查询（对齐后端 M9）

**目标**：对齐奖项模块闭环展示：管理员可发布奖项；学生可查询个人荣誉统计与获奖明细。
**产物**：
- 学生端：我的荣誉页（/me/honors 或嵌入个人中心）
  - participationCount / awardCount / awards 明细列表
  - 空态展示（未参赛/未获奖时显示 EmptyState）
- 管理端：奖项发布入口（可做在竞赛详情的管理区或单独 Awards 管理页）
  - 选择 competition + team + awardName 并提交发布
  - 发布成功提示并展示 recipientCount

**验收**：
- 学生：在获奖后可在“我的荣誉”看到奖项明细；未参赛/未获奖用户显示空态但接口 200；
- 管理员：仅管理员可见“发布奖项”入口；发布成功后可立即用学生端页面验证荣誉更新；
- 对后端 403/409/404 的错误提示统一（尤其是 409：竞赛未结束 / 队伍未锁定 / 队伍已解散 / 重复发布）。

> 风格规则：沿用 F6 统一组件体系，不新增另一套视觉风格。
